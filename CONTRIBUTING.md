# Contributing

Contributions are welcome. Please follow these guidelines to keep changes easy to review and maintain:

- Open an issue before large or cross-cutting changes so we can agree scope and approach.
- Keep changes focused; separate refactors from feature work where practical.
- Follow the existing style and structure; prioritise clarity over cleverness.
- Add or update tests for functional changes and run `pytest` where possible.
- Update documentation (README/CHANGELOG/config examples) when behaviour, outputs, or CLI options change.
- Avoid committing credentials, logs with sensitive data, or large generated artefacts.

By contributing, you agree that your work is licensed under the project's licence (see `LICENSE`).
